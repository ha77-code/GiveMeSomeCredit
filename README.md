💳 信用风险预测项目 —— Give Me Some Credit

本项目基于 Kaggle 经典金融风控数据集 “Give Me Some Credit”，目标是预测借款人在未来两年内是否会发生严重逾期。

项目完整模拟了一个真实的金融风控建模流程，包括数据清洗、探索性分析、特征处理、模型构建、效果评估以及对新客户的风险预测。

📌 项目目标

构建一个 信用违约预测模型，根据用户的历史信用行为和财务状况，预测其未来发生违约的概率。

本项目体现了以下能力：

金融数据预处理

信用风险因素分析

违约预测建模

AUC & ROC 风险模型评估

新客户违约概率预测

🗂 数据集说明

数据来源：Kaggle – Give Me Some Credit

目标变量：

SeriousDlqin2yrs
表示用户是否在未来两年内发生 90 天以上严重逾期
1 = 违约，0 = 未违约

部分核心特征包括：

循环信贷额度使用率

年龄

历史逾期次数（30–59天、60–89天、90天以上）

负债收入比

月收入

信贷账户数量

房产相关贷款数量

家庭赡养人数

⚙️ 项目流程
1️⃣ 数据清洗

对字段进行业务含义标准化重命名

处理缺失值：

月收入（MonthlyIncome） → 使用中位数填充

家庭赡养人数（NumberOfDependents） → 使用中位数填充

删除无实际意义的索引列

2️⃣ 探索性数据分析（EDA）

分析违约样本占比，发现数据存在类别不平衡现象

研究关键变量与违约风险之间的关系

发现高风险特征包括：

高循环信贷额度使用率

高负债收入比

较多历史逾期记录

3️⃣ 特征处理

使用 StandardScaler 对数值特征进行标准化

保持训练与预测阶段数据处理流程一致，确保模型稳定性

4️⃣ 模型构建 —— 逻辑回归

选择逻辑回归的原因：

金融风控领域经典模型，应用广泛

具有良好的可解释性

适合作为信用评分模型的基础

关键操作：

使用分层抽样划分训练集和测试集

通过 class_weight='balanced' 处理违约样本不平衡问题

5️⃣ 模型评估

评估指标包括：

AUC（ROC 曲线下面积） —— 信用风控核心指标

分类报告（精确率、召回率、F1-score）

ROC 曲线可视化

模型能够有效区分高风险客户与低风险客户。

6️⃣ 新客户风险预测

训练完成后，模型可以用于新客户数据：

输出每位客户的 违约概率

识别 高风险客户群体

为风险控制策略提供数据支持

这一步模拟了真实银行或金融机构的风控评分流程。
